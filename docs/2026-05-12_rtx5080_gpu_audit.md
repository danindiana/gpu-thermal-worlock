3. **`nvidia-persistenced`** — keeps the GPU context warm so Ollama inference cold-start is fast.

The RTX 3080 is handling all active screen rendering (Xorg 32% load, alacritty terminals). The RTX 5080 is sitting in P0/idle waiting for the next Ollama inference request.

## Addendum: CUDA_VISIBLE_DEVICES Fix & Tensor-Split Verification (2026-05-12T~1910)

### Problem Found
`/etc/systemd/system/ollama.service.d/override.conf` had `CUDA_VISIBLE_DEVICES=0` — only the RTX 5080 was visible to Ollama's CUDA runtime. This contradicted the intent (both GPUs, tensor-split).

### Fix Applied
```bash
sudo sed -i 's/CUDA_VISIBLE_DEVICES=0"/CUDA_VISIBLE_DEVICES=0,1"/' \
  /etc/systemd/system/ollama.service.d/override.conf
sudo systemctl daemon-reload && sudo systemctl restart ollama.service
```
